# Full Stack C# + GUI Environment Setup (macOS)

## Overview

This document outlines the required tools and steps to set up a complete C# development environment on macOS, including support for GUI development and backend services.

This setup enables:
- C# application development
- Cross-platform GUI development
- Backend API development

---

## Required Toolset

| Category | Tool | Purpose |
|----------|------|--------|
| IDE | JetBrains Rider | Primary development environment for C# |
| Runtime | .NET SDK | Required to build and run C# applications |
| GUI Framework | Avalonia UI | Cross-platform framework for building desktop interfaces |
| API Testing (Optional) | Postman | Tool for testing backend APIs |

---

## Step 1 — Install .NET SDK

Download and install the latest Long-Term Support (LTS) version of the .NET SDK from Microsoft.

After installation, verify that the SDK is available on your system.

---

## Step 2 — Install JetBrains Rider

Download JetBrains Rider for macOS and move it to your Applications folder.

On first launch:
- Sign in with a JetBrains account
- Activate the free non-commercial or student license

---

## Step 3 — Install Avalonia Support

Within Rider:
- Open the Plugins settings
- Search for "Avalonia"
- Install the Avalonia plugin
- Restart Rider after installation

---

## Step 4 — Create a GUI Project

Using Rider:
- Create a new solution
- Select the Avalonia application template
- Initialize the project

This confirms the GUI framework is properly installed.

---

## Step 5 — Run the Application

Use Rider’s run configuration to launch the application.

A window should appear, confirming that:
- The runtime is working
- The GUI framework is functioning correctly

---

## Step 6 — Understand Project Structure

A standard project will include:
- A main window definition (UI layout)
- A code-behind file (logic)
- Application configuration files
- Entry point for application startup

---

## Step 7 — Backend Setup

Install support for backend development using ASP.NET through the .NET SDK.

This allows creation of API services that can be used alongside the GUI application.

---

## Step 8 — API Testing (Optional)

Install Postman or a similar tool to test backend endpoints independently from the GUI.

---

## Common macOS Considerations

- Ensure the .NET SDK is properly added to system paths
- Trust development HTTPS certificates if working with secure APIs
- Restart the IDE if plugins or previews do not load correctly

---

## Final Environment Architecture

The completed setup consists of:
- A desktop GUI application using Avalonia
- A backend API using ASP.NET
- Communication between the two via standard HTTP requests

---

## Summary

This setup provides a complete development environment for building modern C# applications on macOS, including both frontend (GUI) and backend components, using fully supported cross-platform tools.
